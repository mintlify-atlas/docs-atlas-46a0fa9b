<Steps>
  <Step title="Get Telegram API credentials">
    Go to [my.telegram.org/apps](https://my.telegram.org/apps) and sign in with your Telegram account.
    Create a new application to receive your `api_id` (an integer) and `api_hash` (a string).
    You will need both when initializing the `Client`.
  </Step>

  <Step title="Get a bot token from @BotFather">
    Open Telegram and start a chat with [@BotFather](https://t.me/BotFather).
    Send `/newbot`, follow the prompts, and copy the bot token it returns.
    The token looks like `1088394097:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`.
  </Step>

  <Step title="Install Pytdbot">
    Install Pytdbot with the bundled TDLib binary:

    ```bash
    pip install pytdbot[tdjson]
    ```

    This pulls in the `tdjson` wheel which includes a pre-built TDLib shared library.
    See [Installation](/installation) if the wheel is not available for your platform.
  </Step>

  <Step title="Write your first bot">
    Create a file called `bot.py` and paste the following echo bot:

    ```python bot.py
    import logging

    from pytdbot import Client, types

    logging.basicConfig(
        level=logging.INFO,
        format="[%(levelname)s][p %(process)d %(threadName)s][%(created)f][%(filename)s:%(lineno)d][%(funcName)s]  %(message)s",
    )


    client = Client(
        token="YOUR_BOT_TOKEN",
        api_id=0,               # Replace with your api_id
        api_hash="YOUR_API_HASH",
        files_directory="BotDB",
        database_encryption_key="1234echobot$",
        td_verbosity=2,
        td_log=types.LogStreamFile(
            path="tdlib.log", max_file_size=104857600
        ),
    )


    @client.on_updateNewMessage()
    async def print_message(_: Client, message: types.Message):
        print(message)


    @client.on_message()
    async def echo(_: Client, message: types.Message):
        if isinstance(message.content, types.MessageText):
            await message.reply_text(message.text, entities=message.entities)

        elif isinstance(message.content, types.MessageAnimation):
            await message.reply_animation(
                message.remote_file_id,
                caption=message.caption,
                caption_entities=message.entities,
            )

        elif isinstance(message.content, types.MessageAudio):
            await message.reply_audio(
                message.remote_file_id,
                caption=message.caption,
                caption_entities=message.entities,
            )

        elif isinstance(message.content, types.MessageDocument):
            await message.reply_document(
                message.remote_file_id,
                caption=message.caption,
                caption_entities=message.entities,
            )

        elif isinstance(message.content, types.MessagePhoto):
            await message.reply_photo(
                message.remote_file_id,
                caption=message.caption,
                caption_entities=message.entities,
            )

        elif isinstance(message.content, types.MessageSticker):
            await message.reply_sticker(message.remote_file_id)

        elif isinstance(message.content, types.MessageVideo):
            await message.reply_video(
                message.remote_file_id,
                caption=message.caption,
                caption_entities=message.entities,
            )

        elif isinstance(message.content, types.MessageVoiceNote):
            await message.reply_voice(
                message.remote_file_id,
                caption=message.caption,
                caption_entities=message.entities,
            )
        else:
            await message.reply_text("Oops! i don't know how to handle this message.")


    # Run the client
    client.run()
    ```
  </Step>

  <Step title="Run the bot">
    ```bash
    python bot.py
    ```

    Pytdbot will authenticate with Telegram, and your bot will start responding to messages.
    Send any message to your bot and it will echo it back.
  </Step>
</Steps>

## `on_message()` vs `on_updateNewMessage()`

Pytdbot exposes two different decorator patterns for incoming messages:

- **`@client.on_updateNewMessage()`** — fires for the raw TDLib `UpdateNewMessage` object.
  Use this when you need access to the full update envelope, including metadata that wraps the message.

- **`@client.on_message()`** — fires for the unwrapped `Message` object extracted from the update.
  This is the higher-level handler and the one you should use in most cases.

Both can be registered at the same time; they run independently and in the order they are registered.

<Note>
  In the echo bot above, `on_updateNewMessage()` is used only to print the raw update for debugging.
  The actual reply logic lives in `on_message()`.
</Note>
